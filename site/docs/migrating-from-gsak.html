Migrating from GSAK

Step-by-step guide to move your existing GSAK data into GCForge.

Overview

GCForge is designed as the spiritual successor to GSAK (Geocaching Swiss Army Knife). We've made migrating your data as straightforward as possible.

GSAK's concept of separate databases maps to tags in GCForge. Each GSAK database becomes a tag, so you can still organize caches into groups — but now a cache can belong to multiple groups simultaneously.

Preparation

Back Up Your GSAK Database

Before migrating, make a copy of your GSAK database file:

  • Default location: C:\Users\<yourname>\AppData\Local\GSAK\sqlite.db3
  • Or find it in GSAK via File → Database Location

Copy this file to a safe location before proceeding.

What Transfers

GSAK Data GCForge Equivalent Notes
Caches Geocache records All core fields
Database name Tag Each DB becomes a tag
Corrected coordinates Corrected coordinates Direct transfer
Logs Logs Including date, author, type
Waypoints Child waypoints Parking, trailhead, stages
User flags Custom tags Flagged → tagged
Notes Notes field Preserved as-is
Adventure Labs (Q) AL parent + stages Stage order preserved

What Does NOT Transfer

  • GSAK-specific settings and preferences (set up fresh in GCForge)
  • GSAK macros (no equivalent — use GCForge's plugin system in the future)
  • GSAK column layouts (configure in GCForge Settings → Column Presets)

Migration Steps

Step 1: Install GCForge

Follow the installation instructions for your platform.

Step 2: Import the GSAK Database

  1. Launch GCForge
  2. Go to Import → GSAK Database
  3. Select your sqlite.db3 file
  4. Click Import
  5. Wait for the background task to complete

Each GSAK database you imported from will appear as a tag in GCForge. Caches that were in multiple GSAK databases will have multiple tags.

Step 3: Set Your Location

GSAK's "center location" doesn't transfer automatically. Set it up in GCForge:

  1. Go to Import → GSAK locations
  2. Go to Settings → Reference Points
  3. Add your home location coordinates
  4. Assign center points to tags for quick auto-selection
  5. Set one as the default reference point

Step 4: Update Your Finds (Optional)

  1. Download a "All Your Finds" Pocket Query from geocaching.com
  2. Import the GPX in GCForge (merges with existing caches)
  3. Tag these caches as "Found" if desired - but they get an automated flag!

Step 5: Review & Organize

  • Check that all your caches imported correctly
  • Review tags — rename or reorganize as needed
  • Set up column presets for your preferred view
  • Configure your default filters

Troubleshooting

Import is very slow

Large GSAK databases (50,000+ caches) can take several minutes to import. The progress is shown in the task bar. Don't close GCForge during import.

Missing caches

If caches appear to be missing, they may have been filtered out. Reset all filters (Filters → Clear All) and check again.

Corrected coordinates not showing

Make sure you imported from the GSAK database directly (not a GPX export from GSAK). Corrected coordinates are only preserved from the GSAK database import.

Duplicate caches

If you imported the same GSAK database twice, caches are merged by GC code. No duplicates are created — existing records are updated with newer data.
