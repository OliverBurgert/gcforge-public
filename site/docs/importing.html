Importing Data

Import caches from GPX files, GSAK databases, and Adventure Lab exports.

Overview

GCForge supports importing caches from multiple sources. All imported data goes into a single SQLite database — no more juggling separate database files.

Supported import formats:

  • GPX / Pocket Query — standard geocaching.com GPX format with Groundspeak extensions
  • GSAK Database — sqlite.db3 files exported from GSAK
  • Adventure Lab — GPX exports from the lab2gpx tool

GPX / Pocket Query

GCForge imports standard GPX files from geocaching.com, including Pocket Queries downloaded as .gpx or .zip files.

What Gets Imported

  • Cache code, name, type, size, difficulty, terrain
  • Coordinates (posted and corrected)
  • Description and hint (encrypted)
  • Attributes
  • Logs (with dates, authors, and types)
  • Child waypoints (trailhead, parking, stages)
  • Owner information
  • Last found date

Steps

  1. Go to Import → GPX / Pocket Query
  2. Select your .gpx or .zip file
  3. Optionally select tags to apply to the imported caches
  4. Click Import
  5. Import runs in the background — monitor progress in the task bar

Duplicate Handling

If a cache already exists in the database (matched by GC code), the import updates it with the newer data rather than creating a duplicate. Logs are merged — new logs are added, existing logs are not duplicated.

GSAK Database

GCForge can directly import from GSAK's SQLite database files (.db3). This is the recommended migration path for existing GSAK users.

What Gets Imported

  • All GPX fields listed above
  • Corrected coordinates (from GSAK's correction fields)
  • User flags and notes
  • GSAK tags (each database becomes a tag)
  • Child waypoints
  • Adventure Lab stages (from CacheType='Q' rows)

Steps

  1. Go to Import → GSAK Database
  2. Select your sqlite.db3 file
  3. Click Import
  4. The first database import creates tags matching your GSAK database names

Adventure Lab

GCForge imports Adventure Lab GPX exports produced by the lab2gpx tool.

Supported Formats

  • Format A — individual stage waypoints (each stage is a top-level waypoint)
  • Format B — parent adventure cache with child stage waypoints

How It Works

Each Adventure Lab is stored as a parent cache with child stage waypoints. Stage order is determined by the waypoint sequence in the GPX. Adventure identity uses the adventure GUID as the primary key for stability across re-imports.

Background Enrichment

After importing, caches can be enriched with additional data from external sources. Enrichment runs as a background task and includes:

  • Automated fetch of state and county if missing
  • Elevation update

To trigger enrichment: select caches → Actions → Enrich.

Stay up to date

You can update Favorite points and cache status using the light update function on your currently filtered caches. (currently OC only because of missing API key for geocaching.com) → Update → Light Update.
